# AGENTS.md

Guidance for LLM agents working in this repository.

## What this is

Blacklight is a **Ruby on Rails engine gem** (the `blacklight` rubygem) **plus an npm package** (`blacklight-frontend`) providing the Solr discovery UI. The two ship together. Active development for 9.x is on `main`; older series live on `release-8.x` / `release-7.x` branches. Ruby >= 3.3, Rails >= 7.2 and < 9.

## Commands

- **Full CI (what GitHub Actions runs):** `bundle exec rake ci`
- **Lint:** `bundle exec rake rubocop`
- **Default (`rake` with no arg):** runs `rubocop`, then `ci`
- **Single spec file:** `bundle exec rspec spec/path/to_spec.rb` (see testing prerequisites below)
- **Focused example:** add `:focus` metadata (or `fit` / `fdescribe`)
- **Build JS bundles:** `npm run prepare` (rollup → UMD + ESM)
- **Build CSS:** `npm run build` (sass → `app/assets/builds/blacklight.css`)
- **Rebuild everything npm ships:** `bundle exec rake build:npm` (`npm install` + `npm run prepare`)
- **Dev server with Solr:** `bundle exec rake server`
- **i18n checks:** `bundle exec i18n-tasks missing` / `unused`

`rake ci` already chains the correct order: `build:npm` → start Solr → seed → specs with coverage. `rake spec` does **not** start Solr — feature/request/system specs need a live Solr.

## Testing prerequisites (the main gotchas)

- The suite runs **inside a generated Rails app** at `.internal_test_app/` (gitignored), produced by `engine_cart`; `spec/spec_helper.rb` calls `EngineCart.load_application!`. If it doesn't exist, generate it first: `bundle exec rake engine_cart:generate`.
- Specs need **Solr running** with the `analysis-extras` module. `rake ci` / `rake server` start it via docker compose (preferred) or `solr_wrapper`. To point the suite at an already-running Solr, set `SOLR_ENV=docker-compose` (used by `compose.yaml`).
- Feature specs use **Capybara headless Chrome** (`selenium-webdriver`) — Chrome must be installed.
- Seed data is indexed by `rake blacklight:internal:seed` (source: `spec/fixtures/sample_solr_documents.yml`).

### Environment variables that change behavior

- `RAILS_VERSION`, `SOLR_VERSION`, `VIEW_COMPONENT_VERSION`, `BOOTSTRAP_VERSION`, `BUNDLER_VERSION` — select dependency versions (see `.github/matrix.json`).
- `BLACKLIGHT_API_TEST=1` — run only API-mode tests (tagged `:api`).
- `ENGINE_CART_RAILS_OPTIONS` — flags passed to the generated test app (e.g. `--api`, `--css=bootstrap`).
- `COVERAGE=true` — enable SimpleCov (set by `rake blacklight:coverage`).

## Built artifacts — do not hand-edit

- `app/assets/javascripts/blacklight/` — generated by rollup from source in `app/javascript/blacklight-frontend/`. Gitignored, but **shipped in the gem** via `blacklight.gemspec`. If you change JS source, rebuild with `npm run prepare` or the packaged gem will be stale.
- `app/assets/builds/blacklight.css` — generated by sass from `app/assets/stylesheets/blacklight/build.scss`. Rebuild with `npm run build`.

## Architecture

- **Engine entrypoint:** `lib/blacklight.rb` (Zeitwerk autoload; explicitly ignores `generators/`), wired in `lib/blacklight/engine.rb`.
- **Solr adapter:** `lib/blacklight/solr/` (`Repository`, `SearchBuilderBehavior`, `Response`). Repository class is selected from the `adapter` key in `config/blacklight.yml`.
- **`app/`**: `components/` uses **ViewComponent** (a hard runtime dependency), and `app/controllers/concerns/blacklight/catalog.rb` is the core catalog concern. Presenters, services (e.g. `SearchService`), models, helpers, builders, and values round it out.
- **Generators:** `lib/generators/blacklight/install_generator.rb` powers `rails generate blacklight:install`; templates under `lib/generators/blacklight/templates/` are **excluded from RuboCop** — don't lint-fix them blindly.
- **Configuration:** `lib/blacklight/configuration.rb`; defaults registered via `Blacklight::Configuration.initialize_default_configuration` on engine `after_initialize`.

## Style

- RuboCop is large and opinionated (`.rubocop.yml` + `.rubocop_todo.yml`); plugins: `-rails`, `-rspec`, `-rspec_rails`, `-capybara`, `-factory_bot`. Run `bundle exec rake rubocop` before committing.
- Deviations from defaults worth knowing: **line length 200**, **string quote style not enforced** (`Style/StringLiterals` off), **method-def parentheses optional** (`Style/MethodDefParentheses` off), `Naming/PredicatePrefix` forbids `is_` prefixes.
- `i18n_spec.rb` fails on missing keys (unused keys are `pending`); config at `config/i18n-tasks.yml`.
- PR workflow: see `CONTRIBUTING.md`.

## Release

Release the gem and npm package together (see [release wiki](https://github.com/projectblacklight/blacklight/wiki/How-to-release-a-version)). Bump `VERSION` and `package.json#version` **together** (they must match), then:

1. `bundle exec rake release` — builds gem + runs `npm publish` (skip npm with `npm_push=no rake release`).
2. Rebuild derivatives first if needed: `bundle exec rake build:npm`.
